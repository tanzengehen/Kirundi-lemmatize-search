1. Lemmatise

Match the individual words to their keyword (lemma) in the dictionary:
e.g. sort inflected verbs to the infinitive or declined nouns to the nominative singular

   Frequency distribution
Create a table according to the frequency of lemmata used in the text

   Determining the Part of Speech
Mark the Part of Speech of the individual words

   Lemma-soup
Create a text variant in which the individual words are replaced by their lemma

And this is how the Lemmatiser works:

  1. Select the file you want to analyse:
    only txt-files are accepted
  2. Click „Lemmata“
  3. Some statistical values appear in the lower window
    • Number of proper names, adverbs, pronouns, nouns, adjectives, verbs, ...
    • Number of characters in the text
    • Number of tokens in the text (roughly corresponds to the number of words)
    • Number of different words (size of the vocabulary)
    • Percentage of words that the dictionary could not categorise
      These can be spelling mistakes or proper names and words that are not yet listed and should be added to the dictionary as soon as possible
      (you can find out how you can help with this here: Contribute)
    • the version of the dictionary used
  4. There are then four new files in the "results/tagged" folder:
    - Frequency distribution:  fl__dateiname.csv
    - Text labelled with Part of Speech and lemma:  tag__dateiname.csv
    - Normalised text:  norm__dateiname.txt
    - Text variant with lemmata only:  lemma__dateiname.txt

2. Search

When searching in the text, individual words, keywords and word types can be combined.
The search is based on a file in the format tag__dateinname.csv
If you have already labelled the text in the past, it is quicker if you read this tag_file directly.

  1. Select the file you want to search in:
    only txt-files or a tag__filename.csv are accepted
  2. Click "Lemmata":
    tag__dateinname.csv is read or created
    (but this is only necessary if you have searched in another file directly before)
    Some statistical data appears in the lower area
  3. Enter your search query (N-grams)
    The following rules apply:
    • You can search for colourful expressions, i.e. you can combine word types and words and lemmas as you like
    • By default, lemmata are searched for and all associated word forms are collected
    • A / before a /word only allows exact hits for this word
    • A ! directly before a !word explicitly excludes this word (or this lemma or this PoS) from the search
    • A free-standing * is a placeholder for any token at this position. It only applies to words, not to individual letters.
    • The Lemmatiser distinguishes between the following PoS-tags:
    As you enter your search query, you can see in the field below whether the programme has understood you correctly.
  4. Click „Search“
  5. The first 20 text passages that match your query appear in the lower window
    • the number at the beginning indicates the character in the text at which the hit begins
    • a few words before and after the hit are also displayed as context
  6. In the "results/searched" folder, you will find a file in which all the results of this search are saved.
    It is named after your search query: filename_search_terms_.txt
